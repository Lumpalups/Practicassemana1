# Practicassemana1
Caso: Abogabot.

Es un despacho de abogados que quiere automatizar las demandas de sus clientes, por lo que se desarrolla la estructura de una página web en donde el cliente puede llenar un formulario.
esto lo harán a traves de una página web llenando un formulario. Una vez llenado el formulario se realizar el proceso de transacción de pago el cual se le notifica al cliente.
El cliente puede crear una cuenta en la plataforma a través de la cual tendrá seguimiento de su proceso legal con cada actualización del mismo.
El administrador del sitio recbe la notificación de una nueva demanda y con los datos llenados del formulario se crea automaticamente el documento legal en formato word para empezar el proceso.Además el administrador recibe el pago y debe de ser capaz de verlo en un dashboard para ver la cantidad de ingresos recibidos.
Las demandas se van actualizando conforme el proceso legal y se van añadiendo comentarios respectivos.
Al usuario le llegan correos de notificación para saber el avance de su proceso.
Con el objetivo de que el sitio sea responsive será amigable con el teléfono.

USUARIO:
El usuario debe generar una cuenta para poder tener acceso a las actualizaciones. En el home del sitio el usuario tendrá acceso a generar una solicitud o hacer revisión
de la misma. En cada nueva solicitud deberá rellenar un formulario el cual le permitirá al asesor legal tener la información pertinente del usuario y sabra como asistirle.
Una vez generado el formulario el sistema mandara automaticamente el pago generado al usuario.

ADMINISTRADORS:
El administrador tendrá una cuenta especial. El tendrá acceso a las nuevas solicitudes, así como los cambios que se vayan haciendo a los procesos legales en curso. 
Tendra que registrar los avances que su equipo de asesores legales le notifique y debera comunicarlo a traves de comentarios y notificaciones al cliente. Tendra acceso
a los pagos recibidos así como la capacidad de solicitar nuevos pagos para la continuación del proceso legal.

REQUERIMIENTOS:
1.Sign in y sign up
2.Formulario de nueva solicitud
3.Bandeja de notificaciones
4.Buzón de mensajes entre el administrador y el usuario
5.Historial de avances en el proceso legal
6.Sign up de administradores
7.Dashboard de pagos recibidos y procesos legales en curso
8.Dashboard para añadir notificaciones, comentarios y avances en los procesos legales.
9.Automatización de la creación de documentos tipo word una vez llenados los formularios.
10.Buzón de mensajes del administrador donde recibe nuevas solicitudes.
